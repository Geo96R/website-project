### Kubernetes Networking & Ingress

- **Ingress NGINX** - Kubernetes Ingress Controller (Legacy)
  - Website: https://kubernetes.github.io/ingress-nginx/
  - GitHub: https://github.com/kubernetes/ingress-nginx
  - Used for: Legacy ingress routing (kept in k8s-demo for reference)
  - ⚠️ **Retirement Notice**: [Ingress NGINX is being retired in March 2026](https://www.kubernetes.dev/blog/2025/11/12/ingress-nginx-retirement/)
  - Status: No longer receiving updates after March 2026

- **Contour** - High-performance Kubernetes ingress controller
  - Website: https://projectcontour.io/
  - GitHub: https://github.com/projectcontour/contour
  - Documentation: https://projectcontour.io/docs/
  - Used for: Gateway API controller implementation
  - Features: Envoy-powered, Gateway API support, advanced routing, TLS termination
  - Why: Modern, actively maintained, better performance than Nginx

- **Gateway API** - The future of Kubernetes networking
  - Website: https://gateway-api.sigs.k8s.io/
  - GitHub: https://github.com/kubernetes-sigs/gateway-api
  - Documentation: https://gateway-api.sigs.k8s.io/
  - Used for: Modern ingress/routing configuration
  - Features: Role-based API design, better separation of concerns, multi-layer routing
  - Why: Official replacement for Ingress, actively developed by Kubernetes SIG Network

### Observability & Monitoring

- **Grafana Cloud** - Observability platform
  - Website: https://grafana.com/products/cloud/
  - Documentation: https://grafana.com/docs/grafana-cloud/
  - Used for: Metrics, logs, traces, and profiling
  - Features: 
    - Prometheus metrics collection
    - Loki log aggregation
    - OpenTelemetry trace collection
    - Pyroscope continuous profiling
    - Grafana Alloy for data collection
  - Why: Unified observability, managed service, easy integration

- **OpenTelemetry** - Observability framework
  - Website: https://opentelemetry.io/
  - Documentation: https://opentelemetry.io/docs/
  - Used for: Application instrumentation (traces, metrics)
  - Features: Vendor-neutral, auto-instrumentation, standard protocols
  - Integration: Sends data to Grafana Cloud via OTLP
